## Интерфейсы и DI

**Цель:** собрать приложение воедино, используя интерфейсы для Dependency Injection.

- **main** зависит от контрактов (интерфейсов), а не от конкретных пакетов:
  - `BinsStore` — создание bin, список, установка списка
  - `FileReader` — чтение файла, проверка расширения `.json`
  - `BinStorage` — сохранение и загрузка списка bin в/из файла
- Конкретные реализации (`bins.Service`, `file.Service`, `storage.Service`) создаются в `main` и передаются в `run(...)` как интерфейсы.
